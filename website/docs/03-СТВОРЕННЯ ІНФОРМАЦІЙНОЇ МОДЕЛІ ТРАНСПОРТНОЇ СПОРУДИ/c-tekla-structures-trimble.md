## C. Tekla Structures (Trimble)

- ● **Загальний опис:** Tekla Structures є провідним BIM-рішенням для проєктування та деталізації сталевих та залізобетонних конструкцій, включаючи складні мостові споруди. Особливо сильна в деталізації вузлів, випуску робочих креслень КМД (конструкції металеві

ст. 66


деталювальні) та КЗ (конструкції залізобетонні).

*   **Високий рівень деталізації (LOD 400 і вище):** Дозволяє створювати дуже точні та деталізовані моделі, готові для виробництва та монтажу.
*   **Робота зі зварними та болтовими з'єднаннями.**
*   **Інтеграція з виробництвом (CAM).**
*   **Реалізація BIM-концепції:** Об'єктно-орієнтована модель, багата атрибутивна інформація, можливості для координації та співпраці.

### D. Інші спеціалізовані рішення (короткий огляд):

*   **Midas Civil (MIDASoft):** Потужний інтегрований програмний комплекс для проєктування та розрахунку мостів, тунелів та інших інженерних споруд. Має розвинені інструменти для моделювання, аналізу за стадіями зведення, розрахунку на різні типи навантажень.
*   **SOFiSTiK Suite (SOFiSTiK AG):** Комплексне рішення для розрахунку та проєктування будівельних конструкцій, включаючи мости та тунелі. Часто використовується як розрахункове ядро, інтегроване з BIM-платформами (Revit, Rhino/Grasshopper).
*   **Спеціалізовані модулі для тунелів:** Деякі програми (наприклад, ті, що використовуються для геотехнічного аналізу або проєктування гірничих виробок) можуть мати модулі для моделювання тунелів, включаючи елементи обробки, системи вентиляції та безпеки.

Вибір конкретного ПЗ залежить від типу споруди, матеріалу конструкцій, необхідного рівня деталізації, наявних компетенцій у компанії та вимог проєкту.

### 3.4.2. Взаємодія з Іншим Програмним Забезпеченням та Робочі Процеси

Ефективне BIM-проєктування штучних споруд неможливе без налагодженої взаємодії між спеціалізованим ПЗ для моделювання конструкцій та іншими інструментами, що використовуються на різних етапах.

*   **Обмін даними з програмами для проєктування доріг (Civil 3D, Credo, Robur):**
    *   **Геометрія траси:** Ключова інформація (план, поздовжній профіль, поперечні ухили дороги) експортується з дорожнього ПЗ (наприклад, Civil 3D) у форматі **LandXML** або **DWG/DXF** (з 3D-полілініями осі та профілю) та імпортується в програму для моделювання мосту/тунелю (Allplan Bridge, Revit з доповненнями) як основа для створення параметричної моделі споруди.
    *   **Поверхні:** Моделі існуючого рельєфу (ЦММ) та проєктної поверхні дороги також передаються (LandXML, DWG) для коректного розміщення опор, устоїв, визначення висотних відміток, аналізу перетинів.
*   **Використання IFC для координації та інтеграції:**
    *   **IFC (Industry Foundation Classes)** є основним форматом для обміну BIM-моделями між різними програмними платформами.
    *   Модель штучної споруди, створена в Allplan Bridge, Revit або Tekla Structures, експортується в IFC (бажано IFC4 Reference View або IFC4.3 для інфраструктури) для:
        *   **Координації зі зведеною моделлю проєкту:** В програмах типу Autodesk Navisworks, Solibri Model Checker, BIMcollab ZOOM модель мосту/тунелю об'єднується з моделями дороги, інженерних мереж, оточуючої забудови для виявлення колізій та

ст. 67


узгодження проєктних рішень.
- **Передачі замовнику або іншим учасникам проєкту** для перегляду та аналізу.
- **Інтеграції з розрахунковими комплексами** (через IFC Structural Analysis View, якщо підтримується).

* **Зв'язок з AutoCAD для оформлення креслень:**
    * Хоча багато BIM-програм для штучних споруд мають власні інструменти для генерації креслень, часто фінальне доопрацювання та оформлення відповідно до корпоративних стандартів та вимог ДСТУ виконується в AutoCAD.
    * Моделі або 2D-види експортуються з BIM-ПЗ у формат **DWG**. Важливо забезпечити коректну передачу шарів, стилів, анотацій.
    * Використання Простору Листа та Підшивок в AutoCAD для компонування аркушів.

* **Передача моделей у програми для візуалізації (Twinmotion, Infraworks, Blender):**
    * Для створення демонстраційних матеріалів моделі штучних споруд експортуються у формати, що підтримуються програмами візуалізації:
        - **FBX:** Один з найкращих форматів для передачі геометрії, матеріалів (базових), UV-розгорток у Twinmotion, Unreal Engine, Unity, Blender.
        - **IFC:** Може імпортуватися в Twinmotion (через Unreal Engine) або Blender (з BlenderBIM).
        - **Прямі посилання (Direct Link)** або синхронізація (наприклад, між Revit та Twinmotion).
    * Важливо правильно підготувати модель для візуалізації: оптимізувати геометрію, розділити на логічні частини для призначення матеріалів (див. Розділ 7.3).

* **Інтеграція з програмами для перегляду (BIMVision, Solibri Anywhere):**
    * Експортована IFC-модель штучної споруди повинна коректно відкриватися та відображатися в рекомендованих IFC-переглядачах для доступу до геометрії та атрибутивної інформації.

**Типові робочі процеси:**

1. **Дорога -> Міст/Тунель:** Проєктувальник дороги передає геометрію траси та поверхні проєктувальнику штучної споруди.
2. **Моделювання ШС:** Створюється детальна BIM-модель мосту/тунелю в спеціалізованому ПЗ.
3. **Розрахунок ШС:** Аналітична модель передається в розрахунковий комплекс, виконуються розрахунки, результати повертаються для уточнення моделі та армування.
4. **Координація:** IFC-модель ШС інтегрується зі зведеною моделлю проєкту для перевірки на колізії.
5. **Документація:** З BIM-моделі ШС генеруються креслення, специфікації, відомості.
6. **Візуалізація:** Модель ШС передається для створення демонстраційних матеріалів.

Налагоджені робочі процеси та ефективний обмін даними є ключем до успішної реалізації BIM-проєктів штучних споруд.

### 3.4.3. Розрахункова Частина (Структурний Аналіз)

ст. 68


Розрахунок конструкцій мостів, шляхопроводів та тунелів на міцність, стійкість та довговічність є найважливішим етапом проєктування, що забезпечує безпеку та надійність споруд. BIM-технології сприяють більш тісній інтеграції між моделюванням та розрахунками.

### A. Типи розрахунків для мостів, шляхопроводів та тунелів:

*   **Статичні розрахунки:**
    *   На постійні навантаження (власна вага конструкцій, вага дорожнього одягу, тиск ґрунту).
    *   На тимчасові рухомі навантаження (від автотранспорту, поїздів, пішоходів — згідно з нормативними схемами навантажень).
    *   На інші тимчасові навантаження (вітер, сніг, лід, температура, сейсмічні впливи — якщо вони розглядаються як статичні еквіваленти).
    *   Визначення зусиль (згинальні моменти, поперечні та поздовжні сили, крутні моменти), напружень, деформацій, переміщень.
*   **Динамічні розрахунки:**
    *   На рухомі навантаження (з урахуванням динамічного коефіцієнта та коливань).
    *   На вітрові навантаження (пульсаційна складова вітру, аеродинамічна нестійкість).
    *   На сейсмічні впливи (розрахунок за спектральним методом або прямим динамічним методом).
    *   Визначення власних частот та форм коливань конструкції.
*   **Розрахунки на стійкість:**
    *   Загальна стійкість споруди.
    *   Стійкість окремих елементів (стиснутих стійок, балок, арок).
    *   Стійкість укосів та схилів (для тунелів та підходів до мостів).
*   **Розрахунки на втомну міцність:**
    *   Для елементів, що зазнають багаторазових циклічних навантажень (наприклад, від рухомого транспорту).
*   **Розрахунки за стадіями зведення (Construction Stage Analysis):**
    *   Аналіз напружено-деформованого стану конструкції на кожному етапі її будівництва (наприклад, при монтажі прогонових будов, натягуванні вант, бетонуванні окремих секцій). Враховується зміна геометрії, властивостей матеріалів (набір міцності бетоном), прикладання тимчасових монтажних навантажень.
*   **Розрахунки попередньо напружених конструкцій:**
    *   Визначення зусиль попереднього напруження, втрат напруження, напружень та деформацій у бетоні та арматурі.
*   **Геотехнічні розрахунки:**
    *   Розрахунок несучої здатності та осідання фундаментів (пальових, плитних, на природній основі).
    *   Розрахунок стійкості підпірних стін, укосів виїмок та насипів.
    *   Аналіз взаємодії конструкції з ґрунтовою основою.
*   **Гідравлічні та гідрологічні розрахунки (для мостів через водотоки):**
    *   Визначення рівнів води, швидкостей течії, розмивів русла, навантажень від

ст. 69


льодоходу.
- **Розрахунки для тунелів:**
    - Аналіз напружено-деформованого стану обробки тунелю від тиску ґрунту та гідростатичного тиску.
    - Розрахунок стійкості вибою при проходці.
    - Аеродинамічні та вентиляційні розрахунки.

### B. Програмне забезпечення для розрахунків, адаптоване для України:

Вибір розрахункового ПЗ залежить від типу споруди, складності розрахункової схеми, необхідних видів аналізу та, що дуже важливо, підтримки або можливості адаптації до нормативних вимог України.

- **Вбудовані розрахункові модулі в BIM-програмах:**
    - Деякі BIM-платформи для проєктування штучних споруд мають інтегровані або тісно пов'язані розрахункові модулі для попереднього або базового аналізу.
        - **Allplan Bridge:** Може мати зв'язок з розрахунковим ядром Allplan Engineering або інтегруватися з SCIA Engineer (також продукт Nemetschek Group) для більш складних розрахунків.
        - **Autodesk Revit:** Може інтегруватися з Autodesk Robot Structural Analysis Professional для передачі аналітичної моделі та виконання розрахунків.
    - Ці вбудовані можливості часто підходять для попередніх розрахунків, перевірки концептуальних рішень, але для детального аналізу та обґрунтування остаточних проєктних рішень зазвичай використовуються спеціалізовані розрахункові комплекси.

- **Зовнішні спеціалізовані розрахункові комплекси (МСЕ – Метод Скінченних Елементів):**
    - **Ліра-САПР (Lira-SAPR) (Україна):**
        - Широко розповсюджений в Україні програмний комплекс для розрахунку та проєктування будівельних та машинобудівних конструкцій.
        - Має модулі для розрахунку мостів та інших інженерних споруд.
        - **Підтримка українських норм (ДБН):** Реалізовані бібліотеки навантажень, матеріалів, нормативних перевірок згідно з ДБН.
        - Дозволяє виконувати статичні, динамічні, геотехнічні розрахунки, аналіз за стадіями зведення.
    - **SCAD Office (Росія/СНД):**
        - Ще один популярний в країнах СНД розрахунковий комплекс.
        - Також має підтримку багатьох нормативних документів, включаючи ДБН (потребує уточнення актуальності підтримки для останніх версій українських норм).
        - Широкий спектр розрахункових можливостей.
    - **SOFiSTiK Suite (Німеччина):**
        - Дуже потужний та визнаний у світі програмний комплекс для розрахунку складних інженерних споруд, особливо мостів (включаючи вантові, підвісні, попередньо напружені) та тунелів.
        - Базується на МСЕ. Має модульну структуру.
        - Тісна інтеграція з BIM-платформами (Revit, Rhino/Grasshopper, AutoCAD).

ст. 70


*   ■ Підтримує європейські норми (Eurocodes) та багато національних стандартів. Можливість адаптації до українських норм через ручне введення навантажень та параметрів матеріалів, або через розробку спеціальних доповнень (якщо є).
    *   ○ **Midas Civil (Південна Корея):**
        *   ■ Спеціалізований програмний комплекс для розрахунку та проєктування мостів.
        *   ■ Широкий набір інструментів для моделювання різних типів мостів, аналізу за стадіями зведення, розрахунку на рухомі навантаження, сейсміку.
        *   ■ Підтримує багато міжнародних норм. Адаптація до українських ДБН можлива через визначення користувацьких навантажень та матеріалів.
    *   ○ **CSI Bridge (США):**
        *   ■ Продукт від Computers and Structures, Inc. (розробники SAP2000, ETABS). Спеціалізований для аналізу та проєктування мостів.
        *   ■ Підтримка міжнародних норм.
    *   ○ **Загальноінженерні МСЕ-комплекси (Ansys, Abaqus, Nastran):**
        *   ■ Використовуються для дуже складних, нелінійних, наукоємних розрахунків, дослідження специфічних аспектів поведінки конструкцій. Зазвичай не є основними інструментами для масового проєктування, але можуть застосовуватися для унікальних об'єктів або науково-дослідних робіт.

### C. Особливості розрахунків навантажень в Україні:

*   **Нормативна база:**
    *   ○ **ДБН В.1.2-2:2006 "Навантаження і впливи. Норми проектування"** (та його актуальна версія станом на 2025 рік): Основний документ, що регламентує види навантажень (постійні, тимчасові, особливі), їхні значення, коефіцієнти надійності та сполучення.
    *   ○ **ДБН В.2.3-14:2006 "Мости та труби. Правила проектування"** (та його актуальна версія): Встановлює специфічні вимоги до проєктування мостів та труб, включаючи схеми та класи рухомих навантажень (наприклад, НК-80, НК-100, АН, АК).
    *   ○ **ДБН В.2.3-22:2009 "Мости та труби. Основні вимоги проектування"** (та його актуальна версія): Доповнює та уточнює вимоги до проєктування мостів.
    *   ○ **Інші ДБН та ДСТУ:** Що стосуються конкретних матеріалів (бетон, сталь), геотехніки, сейсміки тощо.
    *   ○ **Важливо:** Станом на травень 2025 року ці ДБН могли бути оновлені, замінені або доповнені новими документами, зокрема, у зв'язку з гармонізацією з європейськими нормами (Eurocodes). Завжди необхідно використовувати актуальну чинну нормативну базу.

*   **Специфіка рухомих навантажень:**
    *   ○ Українські норми (зокрема, ДБН В.2.3-14) визначають специфічні класи та схеми рухомих навантажень від автомобільного (АК) та гусеничного (НК) транспорту, а також натовпу (АН), які повинні використовуватися при розрахунках мостів.
    *   ○ Програмне забезпечення повинно дозволяти коректно задавати ці навантаження (або мати вбудовані бібліотеки).

*   **Кліматичні навантаження:**
    *   ○ Снігові та вітрові навантаження визначаються згідно з ДБН В.1.2-2 з

ст. 71


урахуванням кліматичних карт районування України.

*   **Сейсмічні навантаження:**
    *   Для районів із сейсмічністю, визначеною ДБН В.1.1-12 "Будівництво у сейсмічних районах України" (або його актуальною версією), необхідно виконувати розрахунки на сейсмічні впливи.

### D. Взаємодія моделюючого та розрахункового ПЗ:

Ефективний BIM-процес передбачає тісну інтеграцію між програмами для моделювання та програмами для розрахунків.

1.  **Створення аналітичної моделі:**
    *   У BIM-програмі (Allplan Bridge, Revit) на основі фізичної (геометричної) моделі створюється аналітична модель. Вона складається зі спрощених елементів (стрижні, пластини, оболонки), вузлів, граничних умов, інформації про матеріали та перерізи, необхідних для розрахунку.
2.  **Передача аналітичної моделі в розрахунковий комплекс:**
    *   **Прямі посилання (Direct Links / Plugins):** Деякі BIM-програми мають прямі плагіни для зв'язку з певними розрахунковими комплексами (наприклад, Revit та Robot Structural Analysis, Allplan та SCIA Engineer). Це забезпечує найбільш безшовну передачу даних.
    *   **Формати обміну аналітичними моделями:**
        *   **IFC Structural Analysis View (IFC SAV):** Спеціалізований MVD формату IFC, призначений для передачі аналітичної моделі. Підтримка може бути обмеженою.
        *   **SAF (Structural Analysis Format):** Відносно новий відкритий формат на основі Excel, розроблений для обміну даними між програмами для моделювання та розрахунку конструкцій.
        *   **Пропрієтарні формати або API:** Деякі розробники ПЗ пропонують власні формати або інтерфейси програмування (API) для інтеграції.
    *   **Ручне створення розрахункової схеми:** Якщо пряма передача неможлива або дає незадовільні результати, розрахункова схема може створюватися безпосередньо в розрахунковому комплексі на основі геометрії, імпортованої з BIM-моделі (наприклад, через DWG або IFC).
3.  **Виконання розрахунків:**
    *   В розрахунковому комплексі задаються навантаження, виконуються розрахунки, аналізуються результати (зусилля, напруження, деформації, перевірки за нормами).
4.  **Імпорт результатів розрахунків назад у BIM-модель (Round-tripping / Data Exchange):**
    *   **Мета:** Використати результати розрахунків (наприклад, необхідну площу арматури, зусилля в елементах) для подальшого детального проєктування в BIM-середовищі (наприклад, для автоматизованого розкладання арматури, перевірки перерізів).
    *   **Можливості:** Залежать від конкретної пари ПЗ. Деякі інтегровані рішення дозволяють передавати результати назад. В інших випадках дані можуть експортуватися з розрахункового ПЗ у табличному вигляді та використовуватися для ручного коригування.

### 3.4.4. Практичне застосування PythonParts в Allplan Bridge для моделювання елементів мостів: Приклади та переваги автоматизації

Як зазначалося раніше, однією з потужних особливостей Allplan є можливість створення та використання **PythonParts** – параметричних об'єктів, логіка та геометрія яких визначаються за допомогою скриптів на мові програмування Python. Для Allplan Bridge це відкриває значні перспективи для автоматизації моделювання складних та повторюваних елементів мостових конструкцій, а також для адаптації програмного забезпечення до специфічних потреб проєкту або стандартів компанії.

**A. Концепція PythonParts:**

*   **Що це таке?** PythonPart – це інтелектуальний об'єкт в Allplan, який поводиться як стандартний параметричний елемент (наприклад, стіна, колона), але його поведінка, геометрія та атрибути повністю контролюються користувацьким Python-скриптом.
*   **Як це працює?**
    1.  **Розробка скрипту:** Інженер або BIM-спеціаліст (з навичками програмування на Python та знанням API Allplan) пише скрипт, який описує:
        *   **Параметри об'єкта:** Які вхідні дані користувач зможе змінювати (наприклад, висота опори, ширина ригеля, тип перерізу балки, кількість вант).
        *   **Логіку генерації геометрії:** Як на основі вхідних параметрів буде створюватися 3D-геометрія об'єкта (з використанням функцій API Allplan для створення точок, ліній, поверхонь, тіл).
        *   **Призначення атрибутів:** Як будуть розраховуватися та призначатися атрибути об'єкту (матеріал, об'єм, вага, код за класифікатором).
        *   **Інтерфейс користувача:** Як виглядатиме діалогове вікно для введення параметрів PythonPart.
    2.  **Інтеграція в Allplan:** Скрипт зберігається у спеціальному форматі та може бути доданий до бібліотеки об'єктів Allplan.
    3.  **Використання:** Користувач розміщує PythonPart у моделі як звичайний об'єкт, вводить необхідні параметри через діалогове вікно, і скрипт автоматично генерує відповідну геометрію та атрибути. При зміні параметрів об'єкт динамічно оновлюється.

**B. Переваги використання PythonParts для моделювання мостів:**

1.  **Автоматизація створення складних та повторюваних елементів:**
    *   Мости часто містять багато однотипних, але параметрично змінних елементів (опори, сегменти прогонових будов, елементи огородження, деформаційні шви). Створення кожного такого елемента вручну є трудомістким. PythonParts дозволяють автоматизувати їх генерацію.
2.  **Високий рівень параметризації та гнучкості:**

3. **Адаптація до специфічних стандартів та вимог:**

### D. Вимоги та рекомендації для використання PythonParts:

* **Навички програмування на Python:** Для розробки власних PythonParts необхідні знання мови Python та розуміння принципів об'єктно-орієнтованого програмування.
* **Знання Allplan API (Application Programming Interface):** Потрібно вивчити API Allplan, який надає доступ до функцій створення та редагування об'єктів, роботи з атрибутами тощо. Allplan надає відповідну документацію та приклади.
* **Чітке технічне завдання:** Перед розробкою PythonPart необхідно чітко визначити його функціональність, вхідні параметри, очікуваний результат та логіку роботи.
* **Тестування та налагодження:** Розроблені скрипти потребують ретельного тестування та налагодження для забезпечення їх коректної роботи в різних ситуаціях.
* **Документування:** Важливо документувати розроблені PythonParts (опис параметрів, логіки роботи, приклади використання) для зручності їх подальшого використання та підтримки.
* **Поступове впровадження:** Починати з розробки відносно простих PythonParts для автоматизації найбільш рутинних завдань, поступово переходячи до більш складних об'єктів.
* **Навчання та обмін знаннями:** Якщо в компанії є декілька фахівців, що займаються розробкою PythonParts, важливо організувати обмін знаннями та досвідом.
