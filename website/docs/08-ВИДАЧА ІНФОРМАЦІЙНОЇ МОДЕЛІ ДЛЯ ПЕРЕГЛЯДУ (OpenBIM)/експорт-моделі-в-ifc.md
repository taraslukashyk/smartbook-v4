## Експорт Моделі в IFC

Після того, як інформаційна модель транспортної споруди створена, перевірена та насичена необхідною атрибутивною інформацією в основному програмному забезпеченні, наступним кроком є її експорт у відкритий формат IFC. Цей процес вимагає уважного налаштування параметрів експорту для забезпечення максимальної сумісності та збереження всіх необхідних даних.

### 8.3.1. Процес експорту моделі з основного ПЗ (Credo/Civil 3D) у формат IFC

Процедура експорту в IFC залежить від конкретного програмного продукту, що використовується для моделювання.

#### A. Експорт з Autodesk Civil 3D:

Autodesk Civil 3D, як BIM-орієнтована платформа, має вбудовані інструменти для експорту моделей у формат IFC.

1.  **Підготовка моделі до експорту:**
    *   **Видимість об'єктів:** Переконайтеся, що всі об'єкти, які потрібно експортувати (поверхні, траси, профілі, коридори, 3D-тіла коридорів, трубопровідні мережі, об'єкти AutoCAD тощо), є видимими в просторі моделі. Приховані або заморожені шари/об'єкти можуть не експортуватися.
    *   **Об'єкти Civil 3D:**
        *   **Коридори (Corridors):** Для коректного експорту геометрії коридору часто рекомендується створити з нього 3D-тіла (Solids from Corridor) або експортувати поверхні коридору (наприклад, поверхню верху дорожнього одягу, поверхню земляного полотна). Експорт коридору як єдиного об'єкта може бути складним для інтерпретації деякими IFC-переглядачами.
        *   **Трубопровідні мережі (Pipe Networks):** Зазвичай експортуються коректно, якщо труби та колодязі мають 3D-представлення.
        *   **Поверхні (Surfaces):** Експортуються як тріангульовані поверхні (TIN).
    *   **Атрибутивні дані (Property Sets):** Переконайтеся, що до об'єктів прив'язані всі необхідні набори властивостей (Property Sets), і вони заповнені коректними даними.
    *   **Система координат:** Модель повинна мати правильно налаштовану систему координат (див. Розділ 5.2.2).

2.  **Запуск команди експорту в IFC:**
    *   Перейдіть на вкладку "Output" (Вивід) на стрічці.
    *   У панелі "Export" (Експорт) знайдіть команду "Export to IFC" (Експортувати в

IFC) або "IFC Export".
*   ○ Або через меню "Файл" (File) -> "Export" (Експорт) -> "IFC".

**3. Діалогове вікно налаштувань експорту IFC:**
*   ○ **Project Number (Номер проєкту)** та **Project Name (Назва проєкту):** Ці дані можуть бути взяті з властивостей креслення або введені вручну.
*   ○ **File Name (Ім'я файлу)** та **Location (Місце збереження):** Вкажіть ім'я для вихідного IFC-файлу та папку для його збереження.
*   ○ **IFC Version (Версія IFC):** Оберіть потрібну версію IFC (наприклад, IFC2x3, IFC4 Reference View, IFC4 Design Transfer View, IFC4.3, якщо підтримується). **Для цілей перегляду та координації зазвичай рекомендується IFC4 Reference View.**
*   ○ **Model View Definition (MVD):** Автоматично встановлюється залежно від обраної версії IFC (наприклад, Coordination View 2.0 для IFC2x3, Reference View для IFC4).
*   ○ **Exchange requirements / Options (Параметри обміну / Опції):** Натисніть цю кнопку для доступу до більш детальних налаштувань експорту (див. п. 8.3.2).
*   ○ **Objects to Export (Об'єкти для експорту):**
    *   ■ **"Select objects to export" (Обрати об'єкти для експорту):** Дозволяє вручну вибрати об'єкти в кресленні.
    *   ■ **"Export entire drawing" (Експортувати все креслення):** Експортує всі видимі та придатні для експорту об'єкти.
    *   ■ Можливість фільтрації за шарами або типами об'єктів (залежно від версії Civil 3D та налаштувань).

**4. Натискання кнопки "Export" (Експорт).** Процес може зайняти деякий час залежно від розміру та складності моделі.

### B. Експорт з Credo Дороги / Топоматик Robur:

Можливості експорту в IFC у цих програмах можуть відрізнятися залежно від версії та наявності відповідних модулів.

**1. Перевірка підтримки IFC:**
*   ○ Зверніться до документації вашої версії ПЗ Credo або Robur, щоб з'ясувати, чи підтримується експорт в IFC, які версії IFC доступні, та які об'єкти можуть бути експортовані.

**2. Загальний процес (якщо функція доступна):**
*   ○ Зазвичай команда експорту знаходиться в меню "Файл" -> "Експорт" або аналогічному розділі.
*   ○ **Вибір об'єктів:** Може бути можливість експортувати всю модель або вибрані її частини (наприклад, тільки проєктну дорогу, тільки поверхні).
*   ○ **Налаштування параметрів:**
    *   ■ **Версія IFC:** Якщо є вибір, намагайтеся використовувати IFC4 Reference View.
    *   ■ **Система координат:** Важливо, щоб координати експортувалися правильно.
    *   ■ **Атрибути/Семантика:** Перевірте, чи є опції для експорту атрибутивної інформації, пов'язаної з об'єктами.
    *   ■ **Рівень деталізації геометрії (теселяція):** Якщо є налаштування, оберіть

### 3. Збереження IFC-файлу.

**Важливо для Credo/Robur:** Якість експорту в IFC з програм, які історично не були BIM-орієнтованими, може бути нижчою порівняно з Civil 3D або Revit. Геометрія може передаватися не завжди коректно, а атрибутивна інформація може бути обмеженою або відсутньою. Тому обов'язково ретельно тестуйте експортовані IFC-файли в програмах для перегляду. Якщо прямий експорт в IFC дає незадовільні результати, розгляньте альтернативні шляхи, наприклад:

*   Експорт у LandXML (для поверхонь, трас, профілів) з подальшим імпортом в Civil 3D або Blender для створення IFC.
*   Експорт у DWG/DXF з подальшою обробкою та експортом в IFC з AutoCAD-сумісних продуктів.

### 8.3.2. Налаштування параметрів експорту IFC (рівень деталізації, включення атрибутів)

Незалежно від програми, що використовується для експорту, правильне налаштування параметрів є ключовим для отримання якісного IFC-файлу, придатного для перегляду та подальшого використання.

**Версія IFC та Model View Definition (MVD):**

**Як зазначалося, для цілей передачі моделі для перегляду, координації та використання на етапах будівництва та експлуатації, IFC4 Reference View (RV) є найбільш рекомендованим сучасним варіантом. Він забезпечує передачу геометрії та властивостей, необхідних для цих завдань.**

**Якщо потрібна сумісність зі старішим ПЗ, може використовуватися IFC2x3 Coordination View 2.0.**

**Для інфраструктурних об'єктів, якщо ПЗ підтримує IFC4.3, це буде найкращим вибором для точного представлення доріг, мостів тощо.**

**Геометричне представлення та рівень деталізації (Level of Detail - LOD / Tessellation):**

**Теселяція: Криволінійні поверхні (наприклад, поверхні рельєфу, елементи віражів, труби) в IFC зазвичай представляються у вигляді набору трикутників (тесельована геометрія). Налаштування експортера дозволяють контролювати щільність цієї сітки:**

*   **Низька деталізація (Low):** Менше трикутників, менший розмір файлу, але геометрія може виглядати "рубаною".
*   **Середня деталізація (Medium): Розумний компроміс.**
*   **Висока деталізація (High):** Більше трикутників, більш гладка геометрія, але значно більший розмір файлу та можливе сповільнення роботи в програмах для перегляду.

**Обирайте рівень деталізації залежно від призначення IFC-моделі. Для загального перегляду та координації часто достатньо середньої деталізації. Для точних вимірювань або візуалізації може знадобитися вища.**

**Уникайте надмірної деталізації для елементів, які не потребують цього (наприклад, віддалені об'єкти, некритичні деталі).**

### Включення атрибутів (Property Sets / Properties):

**Обов'язково експортуйте атрибутивну інформацію! Саме вона перетворює 3D-геометрію на інформаційну модель.**

**Вибір наборів властивостей: Більшість експортерів (особливо в Civil 3D, Revit) дозволяють обирати, які саме набори властивостей (Property Sets) експортувати.** Включайте всі стандартні та користувацькі набори властивостей, що містять важливу інформацію (матеріали, розміри, типи, коди класифікаторів, посилання на документацію тощо — див. п. 8.5).

**Експорт стандартних властивостей IFC: Багато об'єктів IFC мають стандартний набір властивостей (Pset_Common).** Переконайтеся, що вони також експортуються.

**Кількість атрибутів: Не перевантажуйте модель надлишковими атрибутами, які не будуть використовуватися.** Це може збільшити розмір файлу.

### Система координат та географічна прив'язка:

**Використання спільних координат (Shared Coordinates) або базової точки проєкту (Project Base Point):** Забезпечує правильне позиціонування моделі у світовому просторі та сумісність з іншими моделями (наприклад, при зведенні моделей різних розділів).

**Експорт географічних координат (широта, довгота, висота): Якщо модель має географічну прив'язку, ця інформація повинна бути включена в IFC-файл (зазвичай через об'єкт IfcSite).**

**Уникайте експорту у великих координатах (далеко від початку координат IFC): Деякі програми для перегляду можуть мати проблеми з відображенням геометрії, розташованої дуже далеко від початку координат (0,0,0).** Експортери Civil 3D та Revit зазвичай коректно обробляють це, використовуючи локальне розміщення відносно базової точки проєкту, але зберігаючи інформацію про глобальне положення.

### Структура моделі (Spatial Structure):

IFC підтримує ієрархічну просторову структуру (**Проєкт -> Майданчик -> Будівля -> Поверх -> Приміщення**). Для інфраструктурних об'єктів ця структура може бути адаптована (наприклад, **Проєкт -> Майданчик -> Дорога -> Ділянка**).

**Налаштуйте експорт так, щоб об'єкти були правильно віднесені до відповідних елементів просторової структури.** Це полегшує навігацію та фільтрацію в програмах для перегляду.

### Типи об'єктів IFC (IFC Entity Types):

**Переконайтеся, що об'єкти з вашої моделі (дороги, труби, колодязі, знаки) експортуються з правильними відповідними типами об'єктів IFC** (наприклад, **IfcRoad, IfcPipeSegment, IfcManhole, IfcSign**). Це важливо для семантичної коректності моделі та для можливості фільтрації та аналізу за типами об'єктів. Сучасні експортери (особливо для IFC4.3) повинні це робити автоматично, але іноді потрібне додаткове налаштування мапінгу.

### Розбиття складних об'єктів:

**Як зазначалося в п. 7.3.3, для зручності роботи в програмах перегляду та призначення матеріалів, складні об'єкти (наприклад, дорога з усіма шарами) краще експортувати розбитими на логічні компоненти.** Деякі IFC-експортери мають опції для такого розбиття (наприклад,

ст. 181


"Export parts as building elements" або експорт окремих 3D-тіл).

**Одиниці виміру:** Завжди перевіряйте, що одиниці виміру в IFC-файлі встановлені коректно (зазвичай метри).

**Розмір файлу:** Намагайтеся оптимізувати модель та налаштування експорту так, щоб розмір вихідного IFC-файлу був прийнятним, не втрачаючи при цьому необхідної інформації та точності геометрії. Дуже великі IFC-файли можуть бути складними для обробки деякими програмами.

### 8.3.3. Перевірка коректності експорту

Після завершення процесу експорту в IFC **обов'язково** необхідно перевірити отриманий файл. Не покладайтеся на те, що експорт пройшов ідеально без перевірки.

**Відкриття IFC-файлу в програмі для перегляду:**

Використовуйте одну або декілька програм для перегляду IFC (див. п. 8.4), щоб відкрити експортований файл. Рекомендується перевіряти в тому ПЗ, яке буде використовувати кінцевий користувач моделі, якщо це відомо.

**Візуальна перевірка геометрії:**

*   **Повнота:** Чи всі необхідні об'єкти та елементи присутні в IFC-моделі? Чи немає пропущених частин?
*   **Коректність геометрії:** Чи правильно відображаються форми об'єктів? Чи немає спотворень, розривів, артефактів, "вивернутих" поверхонь?
*   **Взаємне розташування:** Чи правильно розташовані об'єкти відносно один одного та відносно початку координат (якщо це важливо)?
*   **Рівень деталізації:** Чи відповідає деталізація геометрії очікуванням? Чи не надто "рубані" криволінійні поверхні?

**Перевірка атрибутивної інформації:**

Виберіть декілька різних типів об'єктів у програмі для перегляду та перевірте, чи відображаються для них всі необхідні атрибути (Property Sets).

*   Чи коректні значення атрибутів?
*   Чи немає пропущених або порожніх обов'язкових атрибутів?

**Перевірка структури моделі:**

Якщо експортувалася просторова структура (сайт, дорога, ділянки), перевірте, чи відображається вона коректно в навігаторі моделі програми для перегляду.

*   Чи логічно згруповані об'єкти?

**Перевірка системи координат та масштабу:**

Якщо можливо, порівняйте координати характерних точок в IFC-моделі з їхніми координатами у вихідній моделі.

Перевірте загальні габарити моделі, щоб переконатися у відсутності проблем з масштабом.

**Розмір файлу:**

**Чи є розмір отриманого IFC-файлу адекватним для даної моделі? Надто великий розмір може свідчити про надмірну деталізацію або неефективний експорт.**

**Порівняння з вихідною моделлю:**

**Якщо є сумніви, відкрийте вихідну модель та IFC-модель поруч і порівняйте їх візуально та за даними.**

**Якщо під час перевірки виявлені суттєві проблеми, необхідно повернутися до налаштувань експорту в основному ПЗ, внести корективи та повторити експорт. Іноді може знадобитися зміна способу моделювання деяких елементів у вихідній програмі для забезпечення їх кращого експорту в IFC.**
