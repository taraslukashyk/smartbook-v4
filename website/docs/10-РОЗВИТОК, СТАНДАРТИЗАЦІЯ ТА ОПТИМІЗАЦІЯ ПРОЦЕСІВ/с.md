## С

* **САПР (Система Автоматизованого Проєктування) (CAD – Computer-Aided Design)** — Програмне забезпечення, призначене для автоматизації процесів проєктування та створення конструкторської та/або технологічної документації.
* **Синхронізація даних (Data Synchronization)** — Процес узгодження та оновлення даних між двома або більше сховищами або пристроями, щоб забезпечити їхню ідентичність та актуальність.
* **Система координат (Coordinate System)** — Система, що використовується для визначення положення точок у просторі за допомогою числових значень (координат).
* **Спільне Середовище Даних (ССД) (Common Data Environment - CDE)** — Єдине джерело інформації для проєкту, що використовується для збору, управління та поширення документації, графічної моделі та неграфічних даних для всієї проєктної команди.
    * *Джерело: ISO 19650-1:2018.*
* **Специфікація (Specification)** — Документ, що детально описує вимоги до продукції, матеріалів, обладнання або робіт.
* **Стандартизація (Standardization)** — Процес розробки та впровадження стандартів

ст. 231


(правил, норм, вимог) для забезпечення єдності, якості, сумісності та ефективності.
*   **STB (Named Plot Style Table) (в AutoCAD)** — Файл таблиці іменованих стилів друку, де властивості друку (товщина, колір) призначаються об'єктам або шарам через імена стилів, незалежно від їхнього кольору.
    *   *Джерело: Документація Autodesk AutoCAD.*
*   **Стиль (Style) (в AutoCAD, Civil 3D)** — Іменований набір параметрів, що визначає зовнішній вигляд та поведінку об'єктів (наприклад, стиль тексту, розмірний стиль, стиль поверхні, стиль траси).

### Т

*   **Текстура (Texture)** — Растрове зображення, що накладається на поверхню 3D-об'єкта для надання їй більш реалістичного вигляду (імітація матеріалу, візерунку, деталей).
*   **Технічне завдання (ТЗ)** — Див. Завдання на проєктування.
*   **Технічні умови (ТУ) (Technical Specifications / Connection Conditions)** — Документи, що видаються експлуатуючими організаціями та містять вимоги та умови для підключення об'єкта до інженерних мереж або для виконання певних робіт.
*   **Траса (Alignment) (в дорожньому проєктуванні)** — Проєкція осі дороги (або іншого лінійного об'єкта) на горизонтальну площину (план траси) та її висотне положення (поздовжній профіль).
*   **Трасування променів (Path Tracing)** — Метод рендерингу в комп'ютерній графіці, що імітує поширення світла шляхом відстеження шляхів окремих променів від камери або джерел світла. Дозволяє досягти високого рівня фотореалізму.
*   **Twinmotion** — Програмне забезпечення для архітектурної та інфраструктурної візуалізації в реальному часі, розроблене компанією Epic Games.
    *   *Джерело: [https://www.twinmotion.com/](https://www.twinmotion.com/)*

### Ф

*   **Файл (File)** — Іменована сукупність даних, що зберігається на носії інформації.
*   **FBX (.fbx)** — Пропрієтарний формат файлів, розроблений компанією Kaydara (пізніше придбана Autodesk), що використовується для обміну 3D-даними (геометрія, матеріали, текстури, анімація) між різними програмами для 3D-моделювання та ігровими рушіями.
*   **Формат даних (Data Format)** — Структура та спосіб представлення даних у файлі або при передачі.

### Х

*   **Хмарні технології (Cloud Technologies / Cloud Computing)** — Модель забезпечення зручного мережевого доступу на вимогу до спільного пулу конфігурованих обчислювальних ресурсів (наприклад, сервери, сховища даних, додатки), які можуть бути швидко надані та звільнені з мінімальними зусиллями на управління або взаємодію з провайдером.

### Ц

*   **Цифровий двійник (Digital Twin)** — Віртуальна копія фізичного об'єкта, процесу або системи, яка оновлюється даними з реального світу та використовується для аналізу, моделювання, прогнозування та оптимізації.
*   **Цифрова модель місцевості (ЦММ) (Digital Terrain Model - DTM)** — Цифрове представлення поверхні рельєфу місцевості (без урахування рослинності та забудови). Часто

ст. 232


створюється у вигляді тріангуляційної нерегулярної сітки (TIN) або регулярної сітки висот (Grid).

*   **Цифрова модель рельєфу (ЦМР) (Digital Surface Model - DSM)** – Цифрове представлення поверхні місцевості, що включає рельєф, а також об'єкти на ній (будівлі, рослинність).

### Ш

*   **Шаблон (Template)** – Зразок або стандартна форма документа, файлу, моделі або процесу, що використовується як основа для створення нових екземплярів.
*   **Шар (Layer) (в САПР)** – Логічна група об'єктів на кресленні або в моделі, що дозволяє керувати їхньою видимістю, властивостями та організацією.
*   **Штучні споруди (Artificial Structures / Civil Structures)** – Інженерні споруди, що зводяться на транспортних шляхах для подолання природних або штучних перешкод (мости, шляхопроводи, тунелі, естакади, підпірні стінки, труби під насипами).

### E

*   **EIR (Employer's Information Requirements / Вимоги Замовника до інформації)** – Документ, що визначає, яку інформацію, в якому форматі, на яких етапах та для яких цілей Замовник очікує отримати від проєктної команди в рамках BIM-проєкту.
    *   *Джерело: ISO 19650-1:2018.*
*   **Excel (Microsoft Excel)** – Програма для роботи з електронними таблицями, розроблена компанією Microsoft. Широко використовується для розрахунків, аналізу даних, створення відомостей.
    *   *Джерело: [https://www.microsoft.com/uk-ua/microsoft-365/excel](https://www.microsoft.com/uk-ua/microsoft-365/excel)*

### W

*   **Word (Microsoft Word)** – Текстовий процесор, розроблений компанією Microsoft. Використовується для створення та редагування текстових документів.
    *   *Джерело: [https://www.microsoft.com/uk-ua/microsoft-365/word](https://www.microsoft.com/uk-ua/microsoft-365/word)*

### X

*   **XRef (External Reference / Зовнішнє посилання) (в AutoCAD)** – Файл DWG, підключений до поточного креслення як посилання, що дозволяє відображати його вміст без копіювання геометрії.
    *   *Джерело: Документація Autodesk AutoCAD.*
*   **XML (Extensible Markup Language)** – Розширювана мова розмітки, що використовується для зберігання та передачі структурованих даних у текстовому форматі.
