## Використання Корпоративних Шаблонів AutoCAD

Як вже зазначалося, використання корпоративних шаблонів AutoCAD (файлів з розширенням .dwt – Drawing Template) є фундаментальним принципом для стандартизації та підвищення ефективності роботи при створенні робочих креслень. Шаблон містить попередньо налаштовані параметри, стилі та елементи, що дозволяє проєктувальнику одразу розпочати роботу над змістовною частиною креслення, не витрачаючи час на рутинні налаштування.

### 5.2.1. Опис вмісту шаблонів DWG

Корпоративний шаблон AutoCAD для [Назва Вашої Організації/Компанії] повинен бути ретельно розроблений та містити всі необхідні елементи для швидкого та якісного створення креслень транспортних споруд.

**Основні компоненти корпоративного шаблону AutoCAD (.dwt):**

1. **Шари (Layers):**
    * **Розроблена та затверджена корпоративна система шарів:** Кожен шар має унікальне ім'я, призначений колір, тип лінії, товщину лінії, статус друку (друкувати/не друкувати).
    * **Призначення шарів:** Чітко визначено, які типи об'єктів (осі, розміри, тексти, елементи конструкцій, умовні позначення, рамки, штампи тощо) на яких шарах повинні розміщуватися.
    * **Стани шарів (Layer States):** Можуть бути збережені попередньо налаштовані конфігурації видимості шарів для різних типів креслень або завдань.
    * **Приклад структури шарів буде детальніше розглянуто в п. 5.4.1.**

### 2. Типи ліній (Linetypes):
* Завантажені всі необхідні типи ліній згідно з ДСТУ (суцільна, штрихова, штрихпунктирна з двома крапками, осьова, невидимого контуру тощо).
* Можливо, завантажені специфічні корпоративні типи ліній (наприклад, для позначення певних комунікацій).
* Налаштований глобальний масштаб типів ліній (**LTSCALE**) та масштаб типів ліній для простору листа (**PSLTSCALE**).

### 3. Стилі тексту (Text Styles):
* Створені корпоративні стилі тексту для основного тексту, заголовків, розмірних чисел, приміток.
* **Параметри стилю:** Назва стилю, ім'я шрифту (наприклад, "Arial Narrow CYR", "Times New Roman CYR", або спеціалізований креслярський шрифт, що відповідає ДСТУ), висота тексту (часто встановлюється як 0 для можливості задання висоти при створенні тексту, або фіксована висота для анотативних стилів), коефіцієнт стиснення, кут нахилу.
* **Анотативність:** Налаштовані анотативні стилі тексту, які автоматично масштабуються залежно від масштабу видового екрана.

### 4. Розмірні стилі (Dimension Styles):
* Створені корпоративні розмірні стилі, що відповідають вимогам ДСТУ та забезпечують єдиний вигляд розмірів на всіх кресленнях.
* **Параметри стилю:**
    * **Лінії (Lines):** Тип, колір, товщина розмірних та виносних ліній, відступи, подовження за виносні.
    * **Символи та стрілки (Symbols and Arrows):** Тип стрілок (засічки, точки, стрілки), їх розмір.
    * **Текст (Text):** Стиль тексту для розмірних чисел, висота тексту, розміщення відносно розмірної лінії (над лінією, в розриві), вирівнювання.
    * **Розміщення (Fit):** Параметри розміщення тексту та стрілок при недостатньому місці.
    * **Основні одиниці (Primary Units):** Формат одиниць (десяткові, архітектурні), точність (кількість знаків після коми), роздільник цілої та дробової частини, придушення нулів.
    * **Альтернативні одиниці (Alternate Units):** За потреби (наприклад, для відображення розмірів в дюймах).
    * **Допуски (Tolerances):** Якщо використовуються.
* **Анотативність:** Налаштовані анотативні розмірні стилі.

### 5. Стилі мультиліній (Multiline Styles) (якщо використовуються):
* Для креслення стін, доріг або інших об'єктів, що мають паралельні лінії.

### 6. Стилі таблиць (Table Styles):
* Створені корпоративні стилі таблиць для специфікацій, відомостей, експлікацій.
* **Параметри стилю:** Налаштування меж, шрифтів та вирівнювання для комірок даних, заголовків стовпців та назви таблиці.

### 7. Блоки (Blocks):

    - ○ **Одна або декілька попередньо налаштованих вкладок Листів:** З обраним розміром аркуша, підключеним принтером/плотером (або віртуальним принтером, наприклад, "DWG to PDF.pc3"), та встановленим корпоративним стилем друку (CTB або STB).
    - ○ **Налаштовані видові екрани (за потреби):** Наприклад, один видовий екран, що займає основну частину листа.
9. **Налаштування одиниць та системи координат (див. п. 5.2.2).**
10. **Інші налаштування AutoCAD:**
    - ○ **Налаштування прив'язок (Object Snaps).**
    - ○ **Налаштування сітки (Grid) та кроку (Snap)** (зазвичай вимкнені за замовчуванням).
    - ○ **Змінні середовища, що впливають на відображення або поведінку** (наприклад, MIRRTEXT, PICKFIRST).

Ретельно розроблений та підтримуваний в актуальному стані корпоративний шаблон AutoCAD є запорукою високої продуктивності та якості креслярських робіт.

### 5.2.2. Налаштування одиниць та системи координат

Правильне налаштування одиниць виміру та системи координат в шаблоні (та в кожному файлі креслення) є критично важливим для забезпечення точності та сумісності даних, особливо при роботі з інфраструктурними об'єктами, які мають геопросторову прив'язку.

#### А. Одиниці виміру (Units):

*   **Команда UNITS (_UNITS або ОДИНИЦІ):** Відкриває діалогове вікно "Drawing Units" (Одиниці креслення).
*   **Тип довжини (Length Type):** Для інженерних креслень в Україні завжди встановлюється **"Десяткові" (Decimal)**.
*   **Точність довжини (Length Precision):** Кількість знаків після коми. Зазвичай для інфраструктурних об'єктів, що моделюються в метрах, достатньо 2-3 знаки після коми (наприклад, 0.001 або 0.00). Для міліметрів точність може бути 0 або 0.0.
*   **Тип кута (Angle Type):** Зазвичай **"Десяткові градуси" (Decimal Degrees)**. Можливі також "Гради/Мінути/Секунди" (Deg/Min/Sec) або "Радіани" (Radians).
*   **Точність кута (Angle Precision):** Кількість знаків після коми для кутів.

*   **Одиниці вставки (Insertion Scale / Units to scale inserted content):** Дуже важливий параметр! Визначає, в яких одиницях створено поточне креслення. **Для інфраструктурних проєктів, де основною одиницею є метр, тут слід встановлювати "Метри" (Meters).** Якщо креслення виконується в міліметрах (наприклад, для дуже детальних вузлів машинобудівного типу), встановлюється "Міліметри" (Millimeters).
    *   Правильне налаштування одиниць вставки забезпечує коректне масштабування блоків, зовнішніх посилань та інших об'єктів, що вставляються з файлів з іншими одиницями.
*   **Напрямок кута (Angle Direction) та Базовий кут (Angle Base):** Зазвичай залишаються за замовчуванням (0 градусів – на схід, проти годинникової стрілки).

### B. Система координат (Coordinate System):

*   Для інфраструктурних проєктів, які мають географічну прив'язку, необхідно призначити файлу креслення відповідну систему координат. Це особливо важливо при використанні AutoCAD Map 3D або AutoCAD Civil 3D, але корисно і в звичайному AutoCAD для сумісності.
*   **Призначення системи координат (в Civil 3D / Map 3D):**
    *   **Toolspace -> Settings Tab -> Правою кнопкою на імені креслення -> Edit Drawing Settings.**
    *   На вкладці **"Units and Zone"** (Одиниці та Зона) обирається категорія системи координат (наприклад, "Ukraine") та доступна система координат (наприклад, "UCS-2000 / Ukraine Zone 6 (Gauss-Kruger)" або інші зони УСК-2000, або старіші СК-42/СК-63, якщо це вимагається для сумісності зі старими даними).
*   **Переваги призначення системи координат:**
    *   Можливість використання онлайн-карт як підкладки (Geographic Location).
    *   Коректне трансформування даних при імпорті/експорті файлів з іншими системами координат.
    *   Забезпечення геопросторової точності при обміні даними з ГІС-системами.
*   **Якщо використовується звичайний AutoCAD без функцій Map 3D/Civil 3D:**
    *   Пряме призначення геодезичної системи координат може бути відсутнє. У цьому випадку важливо, щоб усі учасники проєкту працювали в узгодженій умовній системі координат, а її параметри (точка початку, орієнтація) були задокументовані.
    *   При обміні даними з іншими системами може знадобитися ручне трансформування координат.

**Важливо:**

*   Налаштування одиниць та системи координат повинні бути виконані в корпоративному шаблоні **.dwt** і перевірятися при створенні кожного нового файлу креслення.
*   При роботі з зовнішніми посиланнями (**XRef**) та блоками переконайтеся, що їхні одиниці вставки узгоджені з одиницями основного креслення.

Правильні налаштування на цьому етапі є запорукою точності вимірювань, коректного масштабування та безпроблемного обміну даними з іншими учасниками проєкту та програмними продуктами.
