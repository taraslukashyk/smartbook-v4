## Створення та Використання Підшивки (Sheet Set)

Підшивка (**Sheet Set Manager / Диспетчер Підшивок**) в AutoCAD — це потужний інструмент для організації, управління, публікації та архівування комплекту аркушів креслень проєкту. Використання Підшивок значно підвищує ефективність роботи з великою кількістю листів, автоматизує багато рутинних завдань та забезпечує узгодженість даних у всьому комплекті.

### 5.6.1. Призначення Підшивки

*   **Організація листів проєкту:**
    *   Підшивка дозволяє логічно структурувати всі аркуші креслень проєкту в єдиному ієрархічному списку, незалежно від того, в скількох файлах DWG вони знаходяться.
    *   Можливість створення підгруп (підшивок) для різних розділів проєкту (наприклад, "Генеральний план", "Автомобільні дороги - План", "Автомобільні дороги - Профілі", "Поперечні профілі").
*   **Централізоване керування властивостями листів:**
    *   Загальні дані проєкту (назва об'єкта, шифр проєкту, замовник, організація-виконавець, ГІП тощо) вводяться один раз у властивостях Підшивки та можуть автоматично відображатися в основних написах (штампах) на всіх аркушах.
    *   Номери та назви аркушів керуються централізовано через Підшивку, що полегшує їх зміну та забезпечує автоматичне оновлення в полях основного напису та в списках аркушів.
*   **Автоматизація нумерації та маркування:**
    *   Автоматична нумерація аркушів у межах підшивки або підгруп.
    *   Можливість створення полів, що посилаються на властивості Підшивки або окремих листів (наприклад, для автоматичного заповнення граф штампу, створення посилань між листами, генерації списків аркушів).
*   **Автоматизація друку та публікації:**
    *   Можливість пакетного друку (публікації) вибраних аркушів, підгруп або всієї Підшивки з використанням попередньо налаштованих параметрів листів (**Page Setups**).
    *   Публікація у формати DWF, DWFx, PDF.
    *   Створення електронних комплектів документації (**eTransmit**).
*   **Створення списків аркушів (Sheet List Tables):**
    *   Автоматична генерація таблиці зі списком аркушів Підшивки, яку можна розмістити на титульному аркуші або в загальних даних. Таблиця динамічно оновлюється при зміні кількості або назв аркушів.
*   **Управління видами та посиланнями:**
    *   Можливість створення іменованих видів у файлах DWG та посилання на них з Підшивки для швидкого переходу або розміщення на аркушах.
*   **Спільна робота:**
    *   Файл даних Підшивки (.dst) може зберігатися на мережевому ресурсі, забезпечуючи доступ для всіх учасників проєкту (з відповідними налаштуваннями прав доступу).

Використання Підшивок є рекомендованою практикою для будь-яких проєктів, що містять більше кількох аркушів креслень.

### 5.6.2. Процес створення Підшивки для проєкту

1. **Запуск Диспетчера Підшивок (Sheet Set Manager):**
    * Команда `SHEETSET` (`_SHEETSET` або `ПОДШИВКА`) або через вкладку **"Application Menu"** (Меню програми) -> **"New"** (Створити) -> **"Sheet Set..."** (Підшивка...).
    * Або через вкладку **"View"** (Вид) -> панель **"Palettes"** (Палітри) -> **"Sheet Set Manager"**.

2. **Створення нової Підшивки:**
    * У Диспетчері Підшивок у випадаючому списку оберіть **"New Sheet Set..."** (Нова підшивка).
    * Запуститься **Майстер створення підшивок (Create Sheet Set Wizard)**.

3. **Етапи роботи Майстра створення підшивок:**
    * **Begin (Початок):**
        * Оберіть опцію **"An example sheet set"** (Зразок підшивки), якщо ви хочете створити підшивку на основі існуючої структури та шаблонів (рекомендовано, якщо є корпоративний шаблон підшивки).
        * Або оберіть **"Existing drawings"** (Існуючі креслення), якщо ви хочете створити підшивку на основі вже наявних файлів DWG з вкладками Листів.
    * **Sheet Set Example (Зразок підшивки) / Sheet Set Details (Відомості про підшивку):**
        * Якщо обрано **"An example sheet set"**, виберіть потрібний шаблон підшивки (наприклад, корпоративний шаблон).
        * Введіть **ім'я нової підшивки** (Name of new sheet set). Це ім'я буде відображатися в Диспетчері.
        * Введіть **опис підшивки** (Description) (опціонально).
        * Вкажіть **місце збереження файлу даних підшивки (.dst)** (Store sheet set data file here). Рекомендується зберігати DST-файл у кореневій папці проєкту або в спеціальній папці для Підшивок.
    * **Sheet Set Properties (Властивості підшивки):**
        * На цьому етапі можна одразу заповнити основні властивості проєкту, які будуть застосовуватися до всіх аркушів:
        * **Project Name (Назва проєкту):** Повна назва об'єкта.
        * **Project Number (Номер проєкту):** Шифр проєкту.
        * **Project Phase (Стадія проєкту):** Наприклад, "РД", "П".
        * Інші користувацькі властивості, визначені в шаблоні підшивки (наприклад, Замовник, ГІП, Організація).
        * Ці властивості потім можна буде використовувати як поля в основних написах.
    * **Confirm (Підтвердження):** Перевірте введені дані та натисніть **"Finish"** (Готово).

Після цього буде створено файл .dst та порожня (або з шаблону) структура Підшивки з'явиться в Диспетчері Підшивок.

### 5.6.3. Додавання листів до Підшивки

Якщо Підшивка створювалася не на основі існуючих креслень, або потрібно додати нові аркуші, це робиться наступним чином:

1. **Імпорт існуючих Листів (Layouts) з файлів DWG:**
    - У Диспетчері Підшивок клацніть правою кнопкою миші на назві Підшивки або на підгрупі та оберіть **"Import Layouts as Sheets..."** (Імпортувати листи як аркуші...).
    - Натисніть **"Browse for Drawings"** (Огляд креслень) та виберіть один або декілька файлів DWG, що містять вкладки Листів, які ви хочете додати.
    - AutoCAD просканує файли та відобразить список доступних Листів.
    - Позначте прапорцями ті Листи, які потрібно імпортувати.
    - Можна обрати опцію **"Prefix sheet titles with file name"** (Додавати ім'я файлу як префікс до назви аркуша), якщо це необхідно.
    - Натисніть **"Import Checked"** (Імпортувати позначені).
    - Імпортовані аркуші з'являться у структурі Підшивки.

2. **Створення нових аркушів у Підшивці:**
    - Клацніть правою кнопкою миші на назві Підшивки або на підгрупі та оберіть **"New Sheet..."** (Новий аркуш...).
    - **Номер аркуша (Number):** Введіть номер нового аркуша.
    - **Назва аркуша (Sheet title):** Введіть назву аркуша (наприклад, "План траси ПК0+00 - ПК10+00").
    - **Шаблон аркуша (Sheet template):** За замовчуванням буде використаний шаблон, вказаний у властивостях Підшивки (якщо є). Можна обрати інший файл DWT.
    - **Ім'я файлу (File name):** Буде запропоновано ім'я нового файлу DWG, в якому буде створено цей аркуш. Можна змінити.
    - **Папка (Folder path):** Вкажіть папку, де буде збережено новий файл DWG.
    - Натисніть **"ОК"**. Буде створено новий файл DWG на основі шаблону, і цей аркуш автоматично додасться до Підшивки.

### 5.6.4. Налаштування властивостей Підшивки та листів

* **Властивості Підшивки (Sheet Set Properties):**
    - Клацніть правою кнопкою миші на назві Підшивки в Диспетчері та оберіть **"Properties..."** (Властивості...).
    - Тут можна редагувати загальні дані проєкту (назва, номер, стадія), які були введені при створенні, а також додавати нові користувацькі властивості (**Custom Properties**).
    - **Sheet creation template (Шаблон для створення аркушів):** Вказується шлях до файлу DWT, який буде використовуватися за замовчуванням при створенні нових аркушів.
    - **Page setup override file (Файл перевизначення параметрів листа):** Можна вказати файл DWT, з якого будуть братися іменовані набори параметрів листів (Page Setups) для застосування до всіх аркушів Підшивки.
    - **Label block for views (Блок мітки для видів):** Налаштування для автоматичного маркування видів.
    - **Callout blocks (Блоки виносок):** Налаштування для блоків, що використовуються для посилань на види та деталі.

*   **Властивості окремого аркуша (Sheet Properties):**
    *   Клацніть правою кнопкою миші на конкретному аркуші в Диспетчері та оберіть "Properties..." (Властивості...).
    *   Тут можна змінити номер аркуша, назву аркуша, опис.
    *   **Layout name (Ім'я вкладки Листа):** Показує ім'я вкладки Листа у відповідному файлі DWG.
    *   **File name (Ім'я файлу):** Показує шлях до файлу DWG, що містить цей аркуш.
    *   **Page setup (Параметри листа):** Можна перевизначити параметри друку для конкретного аркуша, обравши інший іменований набір параметрів.
    *   Користувацькі властивості аркуша.
*   **Використання полів (Fields) для зв'язку з властивостями:**
    *   В основних написах (штампах) на аркушах слід використовувати Поля (команда FIELD) для автоматичного відображення даних з властивостей Підшивки та властивостей аркуша.
    *   Наприклад, для графи "Найменування об'єкта" в штампі створюється поле, що посилається на властивість "Project Name" Підшивки. Для графи "Номер аркуша" — поле, що посилається на властивість "CurrentSheetNumber" аркуша.
    *   Це забезпечує автоматичне оновлення даних у штампах при зміні властивостей в Диспетчері Підшивок.

Використання Підшивок вимагає початкових налаштувань, але значно спрощує управління проєктною документацією в подальшому, особливо для великих та комплексних проєктів.
