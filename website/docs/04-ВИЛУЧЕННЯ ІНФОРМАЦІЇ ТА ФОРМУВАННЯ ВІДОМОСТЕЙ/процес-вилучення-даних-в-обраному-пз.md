## Процес Вилучення Даних в Обраному ПЗ

Процедури та інструменти для вилучення даних відрізняються залежно від використовуваного програмного забезпечення. Розглянемо основні підходи для Autodesk Civil 3D, Credo Дороги та Топоматик Robur.

### 4.2.1. Використання вбудованих функцій звітності в Autodesk Civil 3D

Autodesk Civil 3D надає різноманітні інструменти для вилучення кількісних даних та формування звітів і таблиць.

**A. Quantity Takeoff Manager (Менеджер підрахунку обсягів / QTO):**

*   **Призначення:** Інструмент, призначений для розрахунку обсягів матеріалів та робіт на основі об'єктів креслення AutoCAD та Civil 3D. Дозволяє прив'язувати елементи креслення до статей витрат (Pay Items).
*   **Процес роботи:**
    1.  **Створення списку статей витрат (Pay Item List):** Список може бути імпортований (наприклад, з CSV-файлу, що містить коди та найменування робіт за кошторисними нормами) або створений вручну.

2. **Призначення об'єктів статтям витрат:** Об'єкти креслення (лінії, полілінії, блоки, поверхні, коридори, елементи трубопровідних мереж тощо) вибираються та прив'язуються до відповідних статей витрат.

3. **Визначення методу розрахунку:** Для кожної статті витрат визначається, як буде розраховуватися кількість (наприклад, довжина лінії, площа полігону, об'єм 3D-тіла, кількість блоків).

4. **Генерація звіту:** QTO Manager генерує звіт, який показує обсяги по кожній статті витрат. Звіт може бути експортований у різні формати (HTML, CSV, XML).

* ● **Переваги:** Гнучкість у визначенні статей витрат та методів розрахунку, можливість роботи як з 2D, так і з 3D-об'єктами.
* ● **Обмеження:** Може вимагати значних налаштувань та ручного призначення об'єктів, особливо для складних проєктів. Не завжди інтуїтивно зрозумілий для нових користувачів.

### B. Розрахунок обсягів земляних мас:

* ● **Volumes Dashboard (Панель обсягів):**
    * ○ Основний інструмент для розрахунку обсягів насипу та виїмки між двома поверхнями (зазвичай існуючою ЦММ та проєктною поверхнею, згенерованою з коридору або земляного полотна).
    * ○ Дозволяє створювати декілька записів про обсяги, застосовувати коефіцієнти на ущільнення (для насипу) та розпушення (для виїмки).
    * ○ Генерує зведену таблицю обсягів, яку можна вставити в креслення або експортувати.
    * ○ Можливість створення поверхонь різниць (Volume Surfaces) для візуалізації зон насипу та виїмки.

* ● **Розрахунок обсягів по поперечниках (Compute Materials / Section Views):**
    * ○ Дозволяє розрахувати обсяги матеріалів (включаючи земляні маси та шари дорожнього одягу) на основі аналізу поперечних перерізів, створених вздовж траси.
    * ○ Необхідно визначити критерії підрахунку матеріалів (Material List), де вказується, які об'єкти коридору (форми, ланки) або поверхні відповідають певним матеріалам (наприклад, "Насип", "Виїмка", "Асфальтобетон", "Щебінь").
    * ○ Генерує таблиці обсягів матеріалів, які можна розмістити на аркушах з поперечниками або експортувати.
    * ○ Цей метод часто використовується для детального підрахунку та формування відомостей для кошторисів.

### C. Звіти по об'єктах Civil 3D (Reports Manager):

* ● **Toolbox -> Reports Manager:** Містить велику кількість попередньо налаштованих звітів для різних об'єктів Civil 3D:
    * ○ **Траси (Alignments):** Звіти по геометрії траси, пікетажу, елементах кривих.
    * ○ **Профілі (Profiles):** Звіти по геометрії профілю, вертикальних кривих, відмітках.
    * ○ **Коридори (Corridors):** Звіти по точках коридору, ланках, формах, обсягах коридору (якщо налаштовані матеріали).
    * ○ **Трубопровідні мережі (Pipe Networks):** Звіти по трубах (довжини, ухили, матеріали, діаметри), по колодязях (кількість, типи, відмітки).
    * ○ **Поверхні (Surfaces):** Звіти по обсягах (якщо є поверхня порівняння).

- Можливість налаштування стилів звітів (XSL-файли), але це вимагає знань XML/XSLT.

### D. Таблиці Civil 3D (Tables):

- Багато об'єктів Civil 3D дозволяють створювати динамічні таблиці, які розміщуються безпосередньо в кресленні та автоматично оновлюються при зміні моделі.
    - Таблиці трас (елементи кривих, пікетаж).
    - Таблиці профілів (вертикальні криві).
    - Таблиці обсягів земляних мас (з Volumes Dashboard або з поперечників).
    - Таблиці матеріалів коридору.
    - Таблиці труб та колодязів.
    - Таблиці точок COGO.
- Стилі таблиць можна налаштовувати (шрифти, кольори, стовпці, заголовки).
- Таблиці можуть бути експортовані разом з кресленням (наприклад, у PDF) або їх дані скопійовані в Excel.

### E. Використання наборів властивостей (Property Sets):

- Як зазначалося в Розділі 3.2.4, Property Sets дозволяють додавати користувацькі атрибути до об'єктів.
- Дані з Property Sets можуть бути вилучені та використані для формування звітів та відомостей:
    - **Створення таблиць AutoCAD на основі даних Property Sets:** Можна створити таблицю, яка відображає значення атрибутів для обраних об'єктів.
    - **Експорт даних Property Sets:** Через команду EXPORTPROPSETDATA або за допомогою інструментів, таких як Dynamo, можна експортувати атрибутивні дані в Excel або CSV для подальшої обробки.
    - **Використання в мітках та звітах:** Значення з Property Sets можуть бути включені до складу міток об'єктів або до спеціалізованих звітів, створених за допомогою Reports Manager (якщо звіт підтримує Property Sets) або Dynamo.

### F. Dynamo для Civil 3D:

- **Dynamo** — це інструмент візуального програмування, інтегрований з Civil 3D. Він дозволяє створювати власні скрипти для автоматизації завдань, включаючи вилучення даних.
- За допомогою Dynamo можна:
    - Вибирати об'єкти за складними критеріями.
    - Зчитувати їх стандартні властивості та дані з Property Sets.
    - Виконувати обчислення та перетворення даних.
    - Експортувати дані в Excel, CSV або інші формати у потрібній структурі.
    - Створювати складні звіти, які неможливо отримати стандартними інструментами.
- Використання Dynamo вимагає певних навичок програмування або роботи з візуальними скриптами, але надає надзвичайну гнучкість.
