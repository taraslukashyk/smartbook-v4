### 8.3.2. Налаштування параметрів експорту IFC (рівень деталізації, включення атрибутів)

Незалежно від програми, що використовується для експорту, правильне налаштування параметрів є ключовим для отримання якісного IFC-файлу, придатного для перегляду та подальшого використання.

**Версія IFC та Model View Definition (MVD):**

**Як зазначалося, для цілей передачі моделі для перегляду, координації та використання на етапах будівництва та експлуатації, IFC4 Reference View (RV) є найбільш рекомендованим сучасним варіантом. Він забезпечує передачу геометрії та властивостей, необхідних для цих завдань.**

**Якщо потрібна сумісність зі старішим ПЗ, може використовуватися IFC2x3 Coordination View 2.0.**

**Для інфраструктурних об'єктів, якщо ПЗ підтримує IFC4.3, це буде найкращим вибором для точного представлення доріг, мостів тощо.**

**Геометричне представлення та рівень деталізації (Level of Detail - LOD / Tessellation):**

**Теселяція: Криволінійні поверхні (наприклад, поверхні рельєфу, елементи віражів, труби) в IFC зазвичай представляються у вигляді набору трикутників (тесельована геометрія). Налаштування експортера дозволяють контролювати щільність цієї сітки:**

*   **Низька деталізація (Low):** Менше трикутників, менший розмір файлу, але геометрія може виглядати "рубаною".
*   **Середня деталізація (Medium): Розумний компроміс.**
*   **Висока деталізація (High):** Більше трикутників, більш гладка геометрія, але значно більший розмір файлу та можливе сповільнення роботи в програмах для перегляду.

**Обирайте рівень деталізації залежно від призначення IFC-моделі. Для загального перегляду та координації часто достатньо середньої деталізації. Для точних вимірювань або візуалізації може знадобитися вища.**

**Уникайте надмірної деталізації для елементів, які не потребують цього (наприклад, віддалені об'єкти, некритичні деталі).**
