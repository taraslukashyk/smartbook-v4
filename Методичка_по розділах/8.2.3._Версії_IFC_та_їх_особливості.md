### 8.2.3. Версії IFC та їх особливості

Специфікація IFC постійно розвивається для кращого задоволення потреб будівельної галузі. Існує декілька основних версій IFC, кожна з яких має свої особливості:

* **IFC1.x, IFC2.0, IFC2x, IFC2x2:** Ранні версії, які зараз практично не використовуються.
* **IFC2x3 (випущена у 2006 році):**
    * Довгий час була найпоширенішою та найбільш стабільно підтримуваною версією IFC у більшості програмних продуктів.
    * Має декілька "Model View Definitions" (MVD), таких як Coordination View, Structural Analysis View.
    * Добре підходить для координації моделей будівель, передачі базової геометрії та атрибутів.
    * **Coordination View 2.0 (CV 2.0):** Найбільш часто використовуваний MVD для IFC2x3, призначений для координації між різними дисциплінами.
* **IFC4 (випущена у 2013 році, також відома як ISO 16739:2013):**
    * Значно розширена та покращена версія порівняно з IFC2x3.
    * **Основні нововведення та покращення:**
        * **Нові та покращені класи об'єктів:** Додано багато нових класів для кращого опису різних типів будівельних та інфраструктурних елементів.
        * **Покращена підтримка геометрії:** Більш досконалі способи представлення

складної геометрії (наприклад, NURBS-поверхні, теселяція).
*   **Розширена підтримка інфраструктурних об'єктів (Infrastructure Extensions):** Хоча повноцінна підтримка інфраструктури (дороги, мости, тунелі) активно розвивається в наступних версіях (IFC 4.1, 4.2, 4.3), IFC4 заклав основу для цього.
*   **Нові MVD:**
    *   **IFC4 Reference View (RV):** Призначений для передачі геометрії та властивостей для координації, перегляду, аналізу колізій. Зазвичай забезпечує більш "чисту" геометрію. **Рекомендується для видачі моделі для перегляду та візуалізації.**
    *   **IFC4 Design Transfer View (DTV):** Призначений для передачі моделі з більшою кількістю параметричної інформації та взаємозв'язків між елементами, для можливості подальшого редагування в іншому ПЗ (хоча "round-trip" редагування через IFC все ще є складним завданням).
*   **Покращена робота з властивостями та класифікаторами.**

*   **IFC4.1 (IFC4 Addendum 1, випущена у 2018):**
    *   Включає розширення для інфраструктурних об'єктів, зокрема для мостів (IfcBridge).
*   **IFC4.2 (IFC4 Addendum 2, випущена у 2019):**
    *   Подальший розвиток підтримки мостів та геоТехніки.
*   **IFC4.3 (випущена у 2022 році, також відома як ISO 16739:2022):**
    *   **Значне розширення для інфраструктури:** Включає повноцінну підтримку для доріг (IfcRoad), залізниць (IfcRailway), портів та водних шляхів, тунелів (IfcTunnel). Це дуже важлива версія для інфраструктурних проєктів.
    *   Стандартизація опису трас (alignments), поперечних перерізів, конструктивних шарів.
    *   На момент написання цієї методички (травень 2025 року), підтримка IFC4.3 у комерційних програмних продуктах активно впроваджується, але може бути ще не повною у всіх системах.
*   **Майбутні версії (IFC5 і далі):** Розвиток IFC триває, з фокусом на подальшому розширенні підтримки різних типів об'єктів, покращенні семантичної інформації та інтеграції з іншими стандартами.
